Data Model

The purpose of this data model is for us to have a shared idea of the features and attributes we should collect. Different people like to map different things, and of course you are free to map whatever you want. But hopefully we can make a list of all the features that we'd like to have and we can agree to collect together.

For anyone taking a part of Tbilisi (or elsewhere) on the progress map, you should refer to this data model for a list of things that should be collected.

If you'd like to suggest adding (or removing) something, please let's discuss it on the OSM Georgia Facebook group.

Roads

attribute

tags

notes
road type highway=motorway | primary | secondary | residential | service
name name, name:en, name:ka (name:ru would be nice too but not necessary) At the moment we have agreed that the format for the name field should be "Name in Georgian (Name in English)". For example, "name=ბარნოვის ქუჩა (Barnovi Street)". Please spell out Street, Avenue, ქუჩა, გამზირი, and others. This is the correct method as computers can easily abbreviate these, but it is harder to convert "St." to "Street"
surface type surface=asphalt | cobblestone | unpaved | concrete There are different ways of defining surface type but we should agree on this. Should we do a simple paved|unpaved or should we include the various other types ie. brick, concrete, cobblestone?
width (nice but maybe not necessary) width=(number in meters)
oneway oneway=yes | no One way streets should ALWAYS be marked oneway=yes. It can also be good to mark two-way streets oneway=no, so that we know that someone has looked at the street and determined it is not oneway, rather than nobody having looked at it at all.

Buildings

attribute

tags

notes
identify as building building=yes
number of floors building:levels=(number of stories) building:height is also good. The value should be given in meters. Having one (or both) of these tags is great for 3d rendering programs.
address Each building should have addr:housenumber and addr:street assigned. I think we also need addr:housenumber:en in SOME cases. For example when a building number is 11ა, there should be another tag which says addr:housenumber=11a. I'm not sure but we should probably also have addr:street:en and addr:street:ka

POIs

When it comes to POIs, I think we should just add as many as possible. Of course some POIs may be more important than others, like a school is more significant than a small kiosk, but there's no reason not to map them both. Remember to add tags for
  • name - format is "Georgian Name (English Name)"
  • name:ka
  • name:en
Sometimes places have only an English name or only a Georgian name. My idea is to always use the name:ka field, even if the name is in English letters. If something has no English name, then leave name:en empty. So for example, McDonald's would be:
name=McDonald's
name:ka=McDonald's
name:en=McDonald's
And a "Vulcanizatsia" would be:
name=ვულკანიზაცია
name:ka=ვულკანიზაცია
with no name:en tag.
